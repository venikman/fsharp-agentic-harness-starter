# Execution plan

## Issue

`DEMO-0013`

## Goal

Make file-backed host mode dependency-aware enough to choose runnable issues itself and optionally write tracker state transitions without widening into auto-merge or non-file tracker writes.

## Bounds

- in scope:
  - parse and use `depends_on` for file-backed runtime admission
  - reject manual `run-issue` when prerequisites are unresolved
  - add repo-owned workflow settings for optional file-backed claim/success/failure state transitions
  - leave bounded local host-mode evidence and doc updates
- out of scope:
  - automatic merge or landing of workspace changes
  - writable Linear support
  - continuation-capable multi-turn worker runtime

## Steps

1. Reproduce the current gap: manual dispatch can still target dependency-blocked file issues and host candidate admission ignores `depends_on`.
2. Change the smallest relevant seams: issue parsing, tracker admission, optional file-backed state writes, and host/manual dispatch paths.
3. Validate the repo build and workflow/list surfaces.
4. Run a bounded local host smoke with a temporary file-backed repo to prove dependency ordering plus automatic claim/success transitions.
5. Update repo docs and issue authoring guidance.

## Risks

- risk: writing file-backed states could create dispatch loops if success/failure states stay active
- mitigation: validate workflow config so `tracker.success_state` and `tracker.failure_state` are non-active

- risk: dependency-aware dispatch could imply auto-merge semantics the starter does not actually have
- mitigation: document that admission ordering improves orchestration, but auto-landing remains out of scope
