## Outcome

Host mode now refreshes running issues through the tracker seam on each tick, cancels work that becomes terminal or otherwise non-active, and keeps workspace cleanup policy explicit and test-covered.

## What later work should assume

- tracker refresh during host mode is baseline behavior
- terminal transitions cancel work and optionally clean workspaces according to `workspace.cleanup_terminal`
- non-active, non-terminal transitions cancel work without destructive cleanup
- reconciliation remains tracker-read-only; tracker writes are still out of scope

## Expected deliverables

- per-tick tracker refresh for running issues
- terminal cleanup behavior tied to workflow policy
- non-active stop behavior without cleanup
- deterministic reconciliation tests
- docs that explain the live-run stop rules
