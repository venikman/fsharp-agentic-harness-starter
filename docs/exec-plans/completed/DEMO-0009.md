# Execution plan

## Issue

`DEMO-0009`

## Status

Completed baseline reference retained until plan archival exists.

## Outcome

The tracker seam now supports `tracker.kind: linear` as a read-only GraphQL adapter that preserves the file-backed tracker as the default proving path and keeps required credentials in environment-backed workflow config.

## What later work should assume

- `tracker.project_key` maps to the Linear team key for this adapter
- `tracker.api_key` must be supplied through an environment-variable reference, and `tracker.api_url` defaults to the official Linear GraphQL endpoint
- candidate issue fetch, per-issue refresh, and terminal-issue listing are implemented behind the existing tracker seam
- deterministic validation cover pagination, normalization, team filtering, and GraphQL error propagation without live credentials
- tracker writes such as comments, transitions, and PR links remain intentionally out of scope
- the file-backed tracker still works unchanged for local proving

## Validation

- `dotnet build src/DeliveryHarness.Cli/DeliveryHarness.Cli.fsproj`

## Handoff expectations

A review-ready result should include:
- the workflow/env configuration required for Linear-compatible reads
- confirmation that the file-backed tracker still works
- test evidence for adapter normalization and failure mapping
- clear notes on what remains intentionally out of scope, especially tracker writes
