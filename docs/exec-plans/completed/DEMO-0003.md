# Execution plan

## Issue

`DEMO-0003`

## Status

Completed baseline reference retained until plan archival exists.

## Outcome

`tools/AfterCreate.fsx` and `tools/BeforeRemove.fsx` already implement the git-backed workspace behavior this plan was meant to introduce. New workspaces are provisioned with `git worktree`, reused workspaces are left unchanged by default, and workspace-local `.harness/` stamp files record bootstrap and cleanup outcomes.

## What later work should assume

- git-backed workspace provisioning is baseline behavior, not future work
- workspace reuse/reset policy must stay explicit and conservative
- the hook-driven boundary remains the place for repo-specific bootstrap and cleanup behavior

## Follow-on scope

- strengthen validation or safety around workspace hooks without moving repo-specific provisioning into `DeliveryHarness.Core`
- document any future reuse/reset policy changes in the same change set as the code
