# Execution plan

## Issue

`DEMO-0011`

## Status

Completed baseline reference retained until plan archival exists.

## Outcome

The starter now treats one run record as one attempt, one external worker invocation as one turn, retries as new attempts, and `agent.max_turns > 1` as a named validation failure until a continuation-capable worker runtime exists.

## What later work should assume

- attempt and turn vocabulary is now explicit and documented
- retries increment `AttemptNumber`
- one worker invocation still maps to `TurnNumber = 1`
- run records, status snapshots, and structured logs expose attempt/turn metadata consistently

## Expected deliverables

- honest attempt and turn definitions
- named validation failure for unsupported `agent.max_turns`
- metadata surfaces that expose attempt/turn values
- validation proving the contract

## Follow-on scope

- `DEMO-0012` can rely on the attempt/turn model already being real
- any future continuation-capable worker path should extend this vocabulary rather than replace it silently
