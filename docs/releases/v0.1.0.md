# v0.1.0

## Summary

This release closes the starter harness backlog and proves the trusted-local runtime end to end.

The harness now includes:
- file-backed and read-only Linear tracker intake behind one tracker seam
- deterministic git-worktree workspaces
- long-running host mode with retry/backoff and reconciliation
- structured host logs and JSON status snapshots
- workflow hot reload with last-known-good behavior
- explicit attempt/turn semantics
- strict prompt templating with compatibility mode
- documented issue-state, reload, safety, and recovery contracts

## Highlights

### Orchestration
- `serve` host command
- `status` command
- bounded concurrency in host mode
- retry scheduling from `orchestrator.max_attempts`
- reconciliation against tracker state on each tick
- terminal cleanup policy respected during host mode

### Tracker seam
- file-backed adapter remains the default proving path
- read-only Linear-compatible GraphQL adapter landed for:
  - issue listing
  - candidate fetch
  - per-issue refresh
  - terminal-issue listing

### Workflow contract
- strict `{{ ... }}` prompt templates
- compatibility mode for non-template workflow bodies
- hot reload for future ticks/runs
- restart-required settings explicitly documented

### Observability and evidence
- structured host log at `.harness/runtime/host-events.jsonl`
- JSON status snapshot at `.harness/runtime/status.json`
- structured run records under `.harness/runs/`
