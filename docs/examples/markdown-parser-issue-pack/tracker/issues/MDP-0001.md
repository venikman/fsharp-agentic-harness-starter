# Problem

The markdown parser needs a bounded target before deeper implementation starts. Without an explicit dialect and product boundary, later parsing decisions will drift between undocumented CommonMark assumptions, renderer behavior, and CLI convenience.

# Scope

Define the first supported markdown slice, write down what is intentionally out of scope, and separate parser-library responsibilities from rendering and CLI concerns.
