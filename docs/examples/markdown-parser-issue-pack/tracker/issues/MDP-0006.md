# Problem

Emphasis rules are one of the first places where markdown parsing becomes semantically slippery. Without explicit delimiter rules, the repo will accumulate accidental behavior that is hard to explain or preserve.

# Scope

Implement emphasis and strong emphasis for the chosen dialect slice, with clear documentation of the delimiter rules used for `*` and `_`.
