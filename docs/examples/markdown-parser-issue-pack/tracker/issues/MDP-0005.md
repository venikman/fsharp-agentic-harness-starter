# Problem

Code forms shape both parser semantics and renderer behavior, and they appear often in real markdown. They deserve an explicit bounded step instead of being folded casually into generic text parsing.

# Scope

Implement inline code spans plus indented and fenced code blocks for the initial dialect slice.
