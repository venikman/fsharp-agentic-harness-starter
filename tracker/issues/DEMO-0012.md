# Problem

The current prompt builder prepends issue details and appends the workflow body, which is simple but less expressive than the harness model we want.

# Scope

Add a strict, documented prompt rendering layer for issue context and retry/attempt metadata.

# Notes

Execution plan:
- `docs/exec-plans/active/DEMO-0012.md`

FPF validation:
- lifecycle state: `Shaping`
- admissibility gate: requires `DEMO-0011`
- contract bundle routing: acceptance = promise content, validation = evidence, constraints = admissibility/deontic limits
- vocabulary watch: `prompt template`, `template variable`, `compatibility mode`, `rendered request`

Keep the template surface small and predictable. This is configuration, not a general-purpose programming language.
