# Problem

The prompt-template contract described here has already landed. Workflow bodies now support strict `{{ ... }}` variables for documented issue/attempt/turn fields, and unknown or malformed markers fail validation instead of silently producing misleading prompts.

# Scope

Keep this issue as the historical reference for prompt rendering so later work extends the current small template surface instead of re-describing loose concatenation as the baseline.
