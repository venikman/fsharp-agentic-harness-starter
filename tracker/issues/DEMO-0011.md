# Problem

The attempt/turn contract described here has already landed. The starter now treats one run record as one attempt, one external worker invocation as one turn, retries as new attempts, and `agent.max_turns > 1` as a named validation error until a continuation-capable worker runtime exists.

# Scope

Keep this issue as the historical reference for honest attempt/turn semantics so later work extends the current generic worker model instead of re-describing it as ambiguous.
