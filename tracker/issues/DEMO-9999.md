# Problem

The desired end state is to start the harness once and receive the expected completed project work at the end. The current starter is not yet capable of that because its worker integration, workspace provisioning, orchestration loop, reconciliation, observability, and runtime contract are still being built through the backlog.

# Scope

Define the single program-level bootstrap issue that represents the transition from a starter harness to a one-start autonomous backlog runner for trusted local use.
