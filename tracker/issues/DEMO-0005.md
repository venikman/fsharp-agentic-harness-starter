# Problem

The host-mode scheduler described here has already landed. The starter now exposes `serve`, keeps in-memory running/retrying/retired state, enforces `orchestrator.max_concurrency`, and retries failures with linear backoff based on `orchestrator.poll_interval_seconds`.

# Scope

Keep this issue as the historical reference for host scheduling so later work extends the existing `serve` contract instead of re-describing long-running orchestration as missing.
