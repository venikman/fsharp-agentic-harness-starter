# Problem

Symphony-style harnesses are long-running schedulers. The starter only offers one-shot commands, so retries and continuous dispatch are not yet real runtime behavior.

# Scope

Add a service-host or daemon-style execution mode that owns scheduling, claims, retries, and bounded concurrency.
